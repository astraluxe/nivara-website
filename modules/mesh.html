01 What is Mesh?

Small devices,
enormous models.

Most AI models that actually do useful work — Llama 3.3 70B, Qwen 72B, Mistral Large — need 40–80 GB of RAM. No single laptop has that. Mesh solves this.

Multiple machines on the same WiFi pool their RAM together. Each device holds a portion of the model. Inference runs distributed across all of them. From outside, it behaves like one very powerful machine.

  • Model layers split automatically — no manual setup
  • Built on exo (Apache 2.0, open source)
  • Pure P2P — no cloud, no central server, runs on your hardware
  • Works on MacBook, Windows laptop, Linux desktop — mixed hardware

05 Inter-mesh · paid only

Two meshes,
one supernode.

Paid users become "central nodes" — they own a mesh session and control who joins. Two central nodes can connect their separate meshes, creating one massive unified pool.

  • Only paid central nodes can initiate connections
  • Central node A sends a request to central node B
  • B's user sees an accept/reject prompt
  • Combined pool = A's capacity + B's capacity
Graceful migration
Node A (10 devices) joins Node B (20-device plan, 10 used). If A disconnects: B has 10 free slots → A's devices migrate automatically. Nobody drops.
Hard disconnect
Node A (10 devices) joins Node B (10-device plan, fully used). If A disconnects: B has no free slots → A's 10 devices leave the mesh.

06 How Mesh works

Four steps to
a supernode.

Step 01

Open Models

Open adris.tech desktop app. Go to the Models module. Choose the model you want to run — Llama 70B, Qwen 72B, anything that needs more RAM than you have.

Step 02

Run with Mesh

Click "Run with Mesh" instead of Run. adris.tech starts a node on your machine and generates a room code: adris.tech-472819

Step 03

Share the code

Send the 6-digit room code to friends. They open adris.tech on their machine, click Join Mesh, enter the code — they're in.

Step 04

Model distributes

Layers of the model split across all devices automatically. Each machine holds only its share in RAM. Inference runs distributed, locally — no cloud, no API, no cost.
